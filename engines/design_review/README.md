# Design Review Engine

The Design Review Engine evaluates visual materials such as PPT slides, sales sheets, brochures, white-label sheets, infographics, and HTML prototypes.

## Purpose

Raise visual credibility and B2B decision-maker trust. Design must support business persuasion, not decoration.

## Review Criteria

| Criterion | Review question |
|---|---|
| Message clarity | Is one core message visible within 3 seconds? |
| Hierarchy | Are title, evidence, and supporting details clearly ranked? |
| Alignment | Are boxes, images, text blocks, and margins consistently aligned? |
| Density | Is the page overloaded? |
| Professional tone | Does it avoid childish icons, cliché slogans, and exaggerated wording? |
| Evidence visibility | Are real photos, data, processes, or proof prioritized? |
| Premium B2B fit | Does it feel suitable for enterprise customers and technical reviewers? |

## Default Reviewer Selection

Use Adaptive Persona Engine reviewers:

- P04 Procurement Manager
- P05 Industrial Design Director
- P06 Presentation Reviewer, if slide flow matters
- P10 Red Team

## Design Direction Memory

Preferred design language:

```text
Dyson + SwissRail + Sandvik
```

Default principles:

- minimal icons
- strong alignment
- consistent spacing
- real data or real image first
- no over-marketing
- editable PPT as final format
- HTML as editorial prototype, not final deliverable

## Output Standard

Return:

```text
1. Overall judgment
2. What works
3. What weakens credibility
4. Specific layout fixes
5. Copy-ready revision direction
```
